Getting Started & Installation

Add tinte to your development ecosystem. Verify dependencies and copy-paste standard quickstart boilerplates.

1. Install Package

pip install tinte

2. Basic Script Template

import tinte
import sys

try:
    tinte.intro("Workspace Setup")
    
    username = tinte.text(
        message="What is your username?",
        placeholder="anonymous",
        validate=lambda v: "Required" if not v.strip() else None
    )
    
    tinte.outro(f"Welcome, {username}! 🚀")
except KeyboardInterrupt:
    tinte.cancel("Wizard aborted.")
    sys.exit(0)

1. Add Module

go get github.com/tinte-dev/tinte-go

2. main.go Template

package main

import (
	"fmt"
	"github.com/tinte-dev/tinte-go"
)

func main() {
	tinte.Intro("Go Project Setup")

	name, err := tinte.Text(tinte.TextOptions{
		Message:     "Project name?",
		Placeholder: "my-app",
		Validate: func(s string) string {
			if len(s) == 0 {
				return "Required"
			}
			return ""
		},
	})
	if err == tinte.ErrCancelled {
		tinte.Cancel("Setup aborted.")
		return
	}

	tinte.Outro(fmt.Sprintf("Scaffold completed for %s!", name))
}

Terminal Environment Troubleshooting

Troubleshoot standard terminal issues regarding double-width character layouts, legacy consoles, and UTF-8 encoding variables.

Double-Width CJK Alignment Issues

Wide characters (Korean, Japanese, Chinese) occupy two standard column cells. tinte includes automatic wcwidth calculations, but ensure your terminal uses a CJK-aware font (e.g. D2Coding, NanumGothic Coding, JetBrains Mono) to prevent alignment breaks in Table layouts.

Legacy Windows CMD / PowerShell ANSI support

Legacy Windows consoles (cmd.exe prior to Windows 10) do not interpret ANSI escape sequences by default. tinte handles this natively in Go via console APIs. For Python, ensure colorama is initialized. (tinte imports colorama automatically to resolve this compatibility layer).

Setting UTF-8 Environment Variables

If terminal frames or symbols (│, ╭, ╰) display as garbage text, verify that your active locale is configured to UTF-8:

# Linux / macOS
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8

# Windows (cmd.exe)
chcp 65001
