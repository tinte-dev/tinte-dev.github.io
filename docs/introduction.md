# tinte 🎨: Elegant, Cohesive Terminal Prompts & Displays

`tinte` (German for "ink" or "tint") is a beautiful, themeable, ultra-lightweight terminal prompt and display library designed for **Python** and **Go**. 

Inspired by JavaScript's `@clack/prompts`, `tinte` brings a cohesive, web-like form flow to CLI applications. All components are bound by a signature vertical guide line (`│`), styled with modern pastel palettes, and designed to be dependency-free in Go and lightweight in Python.

---

## 💡 왜 tinte인가? (Motivation)

대부분의 명령줄 도구(CLI)와 설치 마법사는 사용자 인터페이스가 조각나 있고 투박합니다. 
* 화면에 무작위로 출력되는 로그와 입력 프롬프트가 뒤섞여 시각적 흐름이 끊깁니다.
* 입력 값 유효성 검사, 자동 완성, 트리 구조 데이터 표현, 테이블 서식 설정 등을 만들려면 무수히 많은 ANSI 이스케이프 코드와 터미널 이벤트를 직접 작성해야 합니다.

이를 해결하기 위해 JS 생태계에서는 `@clack/prompts`가 등장하여 CLI 디자인의 수준을 높였습니다. 하지만 Python과 Go 개발자들은 다음과 같은 한계에 부딪혔습니다:
1. **무거운 의존성**: JS/Node.js 환경을 반드시 설치해야 하거나, 무거운 TUI 프레임워크(예: Textual, Bubble Tea)를 러닝 커브를 감수하며 배워야 함.
2. **언어 간 파편화**: Python 스크립트와 Go 바이너리로 각각 작성된 내부 CLI 도구들의 UI/UX 디자인이 완전히 제각각임.
3. **복잡한 빌드/설치**: 컴파일된 단일 바이너리(Go)를 제공하려 해도 의존 라이브러리로 인해 실행 환경 제약이 발생함.

**`tinte`는 Python과 Go 모두에서 완전히 동일한 API 사상과 프리미엄 디자인 언어를 단 하나의 외부 라이브러리 추가 없이(Go 기준) 제공하기 위해 탄생했습니다.**

## ✨ 핵심 가치 제안 (Value Propositions)

### 1. 세로 연결선(`│`) 기반의 통합된 인터페이스
모든 프롬프트와 레이아웃 컴포넌트가 세로선(`│`)으로 시각적으로 연결됩니다. 사용자는 여러 단계를 거치는 긴 질문 양식(Form)에서도 단일한 컨텍스트 내에서 진행 상황을 파악할 수 있어, 마치 웹브라우저의 폼을 채우는 듯한 매끄러운 사용자 경험(UX)을 얻습니다.

### 2. Python & Go 듀얼 엔진 탑재
동일한 CLI 디자인 가이드라인이 두 언어에 완벽하게 일치합니다.
* **Python 3.10+**: `colorama`와 `wcwidth`만을 사용하는 가볍고 호환성 높은 구조.
* **Go 1.21+**: 순수 표준 라이브러리(Stdlib)만을 활용한 **제로 의존성(Zero-Dependency)**. 컴파일 속도와 바이너리 크기를 극대화합니다.

### 3. 고도로 설계된 4가지 테마 제공
기본적인 터미널의 투박한 원색(빨강, 파랑 등) 대신, 가독성을 극대화한 현대적 테마를 기본 제공합니다.
* **Pastel (기본)**: 소프트 라벤더와 보라색 톤의 부드럽고 차분한 감성.
* **Ocean**: 시원한 바다를 연상시키는 블루와 민트 컬러.
* **Neon**: 사이버펑크 스타일의 강렬하고 트렌디한 네온 컬러.
* **Mono**: 미니멀하고 깨끗한 그레이스케일.

### 4. 강력한 대화형 컴포넌트 & 레이아웃 내장
단순 입력을 넘어, 복잡한 데이터를 출력하고 탐색할 수 있는 컴포넌트들을 제공합니다.
* **인터랙티브 트리(Tree)**: JSON이나 Dict 객체를 터미널에서 위/아래 방향키와 스페이스바로 직접 열고 닫을 수 있습니다.
* **인터랙티브 아코디언(Accordion)**: 길고 복잡한 설정 가이드나 약관 등을 아코디언 메뉴로 감싸 터미널 화면 낭비를 막습니다.
* **CJK 문자폭 교정**: 한국어, 일본어, 중국어 등 동아시아 문자가 테이블이나 레이아웃에서 깨지거나 정렬이 어긋나는 문제를 내부 엔진(`wcwidth` 호환 로직)으로 완벽하게 제어합니다.

## ⚡ 기대 효과 (Concrete Benefits)

### 🛠️ 개발 생산성 극대화 (DX)
터미널 UI 렌더링, 정렬, 포커싱, 입력 루프 등을 구현하려면 파일당 수백 줄의 코드가 필요합니다. `tinte`를 사용하면 단 몇 줄의 직관적인 선언형 코드로 동일한 기능을 구현할 수 있어 비즈니스 로직 작성에만 집중할 수 있습니다.

### 📉 사용자의 CLI 작업 이탈률(Drop-off) 감소
복잡한 인프라 설정 스크립트나 로컬 개발 도구 설치 시, 유효성 검사 미흡이나 경로 입력 오류로 도중에 실패하는 경우가 많습니다. `tinte`는 실시간 유효성 검사, 디렉토리 탭 자동 완성, 에러 메시지 툴팁을 제공하여 입력 실수를 차단합니다.

### 💎 소프트웨어 브랜딩 가치 제고
사용자 또는 고객이 처음 마주하는 설치 화면이나 CLI 마법사가 세련될수록, 서비스 전체의 완성도와 브랜드 인지도에 대한 신뢰성이 급상승합니다.

## 🧩 제공되는 컴포넌트 목록 (Full UI Components)

### 7가지 인터랙티브 입력 프롬프트 (Interactive Prompts)
1. **`text`**: 사용자 정의 유효성 검사(Validate) 및 플레이스홀더를 제공하는 기본 문자열 입력.
2. **`password`**: 입력된 글자를 숨겨주고 강도 체크를 할 수 있는 보안 입력.
3. **`select`**: 방향키로 하나만 고를 수 있는 단일 선택 메뉴.
4. **`multiselect`**: 여러 개를 고르고 개수 제한을 둘 수 있는 다중 선택 메뉴.
5. **`confirm`**: 예/아니오(Y/N) 신속 응답 단축키 지원 프롬프트.
6. **`autocomplete`**: 타이핑에 따라 실시간으로 옵션을 필터링하는 퍼지 검색 검색창.
7. **`path`**: 파일/폴더 자동 완성 및 존재 여부 검사를 지원하는 파일 시스템 탐색기.

### 9가지 시각적 레이아웃 컴포넌트 (Visual Displays)
1. **`table`**: 네 가지 테두리 스타일(둥근 모서리, 각진 모서리, 이중선, 최소선)을 지원하는 정렬식 테이블.
2. **`tree` (대화형)**: 복잡한 JSON 데이터를 아코디언처럼 펼쳐서 폴더 구조로 탐색할 수 있는 시각화 위젯.
3. **`accordion` (대화형)**: 여러 텍스트 콘텐츠 단락을 터미널 내에서 접고 펼칠 수 있는 툴셋.
4. **`panel`**: 코드 가이드나 팁을 표시하기 위해 외곽선을 감싼 정보 상자.
5. **`note`**: 중요 사항이나 웰컴 메시지를 눈에 띄게 배치하는 타일식 메모.
6. **`diff`**: 변경 코드나 구성 파일 차이점을 빨간색/초록색으로 명확히 보여주는 통합 디프 뷰어.
7. **`progress`**: 긴 작업을 위한 부드러운 진행률 표시 바.
8. **`syntax`**: 소스 코드를 구문 강조하여 화면에 안전하게 렌더링하는 구문 강조 도구.
9. **`keyvalue`**: 설정된 환경 변수나 메타데이터를 정밀 정렬하여 표시하는 키-값 쌍 리스트.
